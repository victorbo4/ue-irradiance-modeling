Pyrano – Capture Planner

Pyrano automates solar irradiance capturing in Unreal Engine using sensors and time-based simulation. This documentation covers how to configure, run and export data properly.

1. Scene Setup

To use Pyrano, make sure your level contains:

  • SunSky — comes with official plugin SunPosition.
  • PyranometerComponent — Add it to any actor.
  • Enable sensors you want to simulate (bEnabled = true). You can also enable them in the Pyranometer's tab.

2. Planner Configuration

The Capture Planner window allows you to define:

  • Latitude / Longitude — Geographic coordinates.
  • Timezone — Local offset relative to UTC.
  • North Offset - Adjusts SunSky orientation so your map's north matches real-world true north. Use it when your scene is not oriented to real-world north.
  • Start Time / End Time — Simulation range.
  • Sample Interval — Time step between captures.
  • Resolution Px — Cubemap size (256 or 512 recommended).
  • Path Tracing — More accurate, but slower. Path Tracing may not be supported on all PCs.
  • Export CSV — Saves irradiance results to disk.

3. Running Captures

Capture Once

Runs a single irradiance measurement for every enabled sensor, using Start Time as the timestamp.

Start Simulation

Runs all sensors from Start Time to End Time, advancing by Sample Interval.

Pyrano will:

  • Open a dedicated PIE window.
  • Run captures automatically.
  • Close the window when finished.

4. Tips & Good Practice

  • Do not close the PIE window manually — the plugin handles it.
  • Make sure that at least one sensor is enabled.
  • If exporting CSV, make sure the output path is valid.
  • Higher cubemap resolution increases time, but doesn’t significantly affect irradiance accuracy.
  • It is recommended to split long simulations into smaller ones.
